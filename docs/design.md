# VoxCanvas Lab 设计文档

## 题目选择

第四批次题目二：AI 语音绘图工具。

要求是开发一款纯语音控制的绘图工具，用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作。需要综合考虑指令理解准确性与容错性、语音到绘图操作的响应延迟，以及复杂指令的拆解与执行能力。还需要记录计划支持的指令能力、最终实现的能力，以及未完成部分的原因说明。

## 用户故事

计划实现：

1. 用户可以打开麦克风，通过一句自然语言完成绘图。
2. 用户可以说出颜色、形状、数量和位置，系统自动绘制。
3. 用户可以说出复杂场景，系统自动拆解成多个绘图动作。
4. 用户可以用语音清空、撤销、重做、导出图片。
5. 用户可以看到系统如何理解指令，包括动作队列、置信度和延迟。
6. 系统默认不调用云端模型，降低运营成本。

最终实现：

1. 已实现麦克风启动与浏览器 Web Speech API 语音识别。
2. 已实现本地语义解析器，支持颜色、形状、数字、位置、场景关键词。
3. 已实现复杂指令拆解，例如夕阳、夜景、森林、山脉、河流、房子、星星等组合。
4. 已实现 Canvas 绘图执行器，包括背景、圆形、矩形、三角形、线条、箭头、星星、爱心、树、房子、笑脸、太阳、月亮、云朵、花朵、文字。
5. 已实现本地语音反馈、动作列表、事件日志、解析置信度、延迟显示和 PNG 导出。
6. 已实现 parser smoke test，验证主要指令能映射到预期动作。

## 指令能力设计

计划支持：

- 基础绘图：circle、rectangle、triangle、line、arrow、star、heart、cloud、flower、text。
- 场景绘图：sunset、night、forest、mountain、river、house、sky。
- 属性控制：color、size、radius、position、count。
- 编辑控制：clear、undo、redo、save/export。
- 容错能力：颜色别名、数字词、中文关键词、英文同义词。

最终实现：

- 基础绘图：已完成。
- 场景绘图：已完成，复杂场景会拆解成多个动作。
- 属性控制：已完成常用颜色、数字、位置和尺寸解析。
- 编辑控制：已完成清空、撤销、重做、导出。
- 容错能力：已完成规则化容错。未接入云端大模型，因此超长开放式描述会退化为抽象标记或部分场景拆解。

## 复杂指令拆解

示例：

```text
画一个蓝天背景，有三朵白云和四朵粉色花
```

拆解为：

1. background: sky
2. cloud x 3: upper area
3. flower x 4: bottom row

示例：

```text
draw a sunset background with mountains and a river
```

拆解为：

1. background: sunset
2. sun: upper right
3. mountains: bottom skyline
4. river: lower curve

示例：

```text
make a forest with three trees and a moon
```

拆解为：

1. background: night
2. moon: upper right
3. stars: generated across upper area
4. tree x 3: bottom row

## 延迟策略

语音到绘图链路分为三段：

1. 浏览器语音识别：使用 Web Speech API，避免把音频传到自建服务。
2. 本地语义解析：`app-core.mjs` 纯前端同步执行，通常毫秒级。
3. Canvas 渲染：所有绘制动作在浏览器本地完成。

页面会展示每次指令的解析与执行延迟。实际延迟主要取决于浏览器语音识别返回 final transcript 的速度。

## 成本控制

想到的技巧：

1. 默认使用浏览器语音识别，不自建 ASR 服务。
2. 绘图语义解析使用本地规则，不把每条语音都发给大模型。
3. 将复杂指令拆成结构化动作后一次性执行，避免多轮云端请求。
4. 页面中显示 `0 cloud calls`，把成本暴露给用户和开发者。
5. 后续若接入大模型，只在本地解析失败时触发 fallback，并设置请求频率、token 上限和缓存。

实际采用：

1. 采用纯前端本地解析与 Canvas 渲染。
2. 不上传音频、画布和历史命令。
3. 不调用云端模型，因此演示和评测时理论云成本为 0。

## 未完成与原因

1. 未实现任意自然语言到任意图像的开放式生成。原因是题目强调绘图操作控制，本作品优先保证可解释、低成本、低延迟，而不是调用图像生成模型。
2. 未实现跨浏览器统一语音识别。Safari/Firefox 对 Web Speech API 支持有限，当前建议使用 Chrome 或 Edge。
3. 未实现多人协作画布。原因是 72 小时作品挑战中，优先级低于语音链路、指令拆解和成本控制。
4. 未实现完全无点击启动麦克风。浏览器安全策略要求用户手势触发麦克风权限，因此保留一个启动按钮。

## 文件说明

- `app-core.mjs`：语义解析和指令拆解。
- `app.js`：语音识别、Canvas 绘制、状态管理和 UI 绑定。
- `styles.css`：响应式界面与视觉风格。
- `tests/parser-smoke.mjs`：解析器基本测试。
